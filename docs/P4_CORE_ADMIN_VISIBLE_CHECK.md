# P4 Core Admin Visible Check

更新时间：2026-06-03

## 目标

P4.6 把工作台从模块工具入口扩展为核心后台页面验收入口，确保用户进入后台后能看到并打开通用管理框架最基础的页面。

## 当前落地

- `GET /api/common/index/console` 返回 `corePages`。
- 工作台新增 `核心页面验收` 表格。
- 工作台阶段更新为 `P4.6 核心管理页可见验收`。
- 工作台验收状态新增：
  - `模块中心`
  - `核心页面入口`
- 核心页面验收表格包含：
  - 菜单权限：`/menu`
  - 角色管理：`/role`
  - 管理员：`/admin`
  - 组织部门：`/department`
  - 网站信息：`/information`
  - 系统缓存：`/cache`
  - 系统日志：`/journal`

## 人工测试

本阶段已在浏览器完成：

- 打开 `http://127.0.0.1:5173/workbench`。
- 工作台显示 `核心页面验收` 和 `P4.6`。
- 打开 `/menu`，页面显示 `菜单名称`。
- 打开 `/role`，页面显示 `管理员人数`。
- 打开 `/admin`，页面显示 `管理员账号`。
- 打开 `/department`，页面显示 `部门名称`。
- 打开 `/information`，页面显示 `网站名称`。
- 打开 `/cache`，页面显示 `基本信息`。
- 打开 `/journal`，页面显示 `访问链接`。
- 上述路由均未出现 403、404 或空白页。
