# P5 Module Status Filters

更新时间：2026-06-03

## 目标

P5.4 把模块中心的安装状态回读从单行展示推进到可筛选、可汇总的模块列表形态，为后续接入多个业务模块做准备。

本阶段不新增后端写入，不创建业务 schema。

## 管理端改动

模块中心内置模块清单新增：

- 状态筛选：`全部`、`已安装`、`部分安装`、`未安装`、`异常`。
- 状态汇总：总数、已安装、部分、未安装、异常。
- 筛选空态：无匹配模块时显示 `暂无匹配模块`。

状态数据仍来自 P5.3 的只读接口：

```text
POST /api/gen/previewCode/status
```

## 验收结果

- 已通过 `cd admin && npm run type-check`。
- 已通过浏览器人工验证 `/module` 显示 `P5.4`、状态汇总和 Demo Article 的 `已安装` 状态。
- 已通过浏览器人工验证 `未安装` 筛选显示 `暂无匹配模块`。
- 已通过浏览器人工验证切回 `全部` 后恢复 Demo Article 行，并显示 `权限 5/5` 和 `角色授权 5/5`。

## 保留边界

P5.4 不做：

- 不新增模块 registry 后端列表接口。
- 不创建或迁移 `ma_demo_article`。
- 不把写入 env 写进 `.env`。
- 不连接 zyai 真实业务库。
- 不处理 PTLM 业务模块。
