# P4 Module Center Status Checklist

更新时间：2026-06-03

## 目标

P4.5 把模块中心从操作入口推进为验收工作台：用户生成模块预览后，可以直接看到模块状态、写入门禁状态和人工测试清单。

## 当前落地

- `admin/src/views/dev_tools/module/index.vue` 新增 `模块状态`。
- 模块状态展示：
  - 预览状态
  - 安装状态
  - 卸载状态
  - 写入门禁
  - 运行时开关
- 新增 `人工测试清单`。
- 人工测试清单展示：
  - Manifest 预览
  - 安装计划
  - 代码预览
  - 安装执行
  - 卸载执行
  - 审计预览
- 打开安装计划后，清单会标记 `安装计划` 为 `已打开`。
- 打开代码预览后，清单会标记 `代码预览` 为 `已打开`，并显示生成文件数量。
- install/uninstall apply 返回后，清单会显示当前 apply 状态和结果说明。
- 写入门禁环境变量以可换行 tag 展示，避免长环境变量挤压页面。
- 运行时开关和表格长文本支持换行。

## 人工测试

本阶段已在浏览器完成：

- 打开 `http://127.0.0.1:5173/module`。
- 生成 `examples/demo/manifest.json` 预览。
- 页面显示 `模块状态` 和 `人工测试清单`。
- 打开 `安装计划`，清单标记为 `已打开`。
- 打开 `代码预览`，清单标记为 `已打开`，并显示生成文件数量。
- 勾选 `安装写入` 后点击 `安装执行`，页面显示 blocked 状态和 `MAKEADMIN_ALLOW_MODULE_INSTALL_APPLY`。
- 勾选 `删除确认` 后点击 `卸载执行`，页面显示 blocked 状态和 `MAKEADMIN_ALLOW_MODULE_UNINSTALL_APPLY`。
- 浏览器验证写入门禁 env tag、运行时开关和描述 label 样式均可换行或保持正常宽度。

## 当前边界

P4.5 不做：

- 不新增后端状态探测接口。
- 不开启安装/卸载写入环境变量。
- 不实际写入或删除本地模块菜单权限。
- 不新增数据库 schema。
- 不修改 `.env`。
- 不连接真实 zyai 业务库。
